# Виконання роботи

## 1. Робота з PIP та сторонніми бібліотеками

Було перевірено наявність менеджера пакетів `pip` та переглянуто список доступних команд за допомогою:

```bash
pip -V
pip --help
```

Також було переглянуто доступні команди `pipenv`:

```bash
pipenv --help
```

### Основні команди pipenv:

* `install` — встановлення бібліотек;
* `remove` — видалення середовища;
* `upgrade` — оновлення пакетів;
* `uninstall` — видалення бібліотеки;
* `lock` — створення `Pipfile.lock`;
* `shell` — запуск shell у середовищі;
* `run` — запуск команд у середовищі;
* `graph` — дерево залежностей;
* `check` — перевірка вразливостей;
* `audit` — аудит безпеки пакетів.

## 2. Встановлення та використання бібліотеки requests

Було встановлено бібліотеку `requests` та протестовано її роботу.

Приклад використання:

```python
import requests

response = requests.get('https://httpbin.org/')
for line in response.iter_lines():
    print(line)
```

```bash

b'<!DOCTYPE html>'
b'<html lang="en">'
b''
b'<head>'
b'    <meta charset="UTF-8">'
b'    <title>httpbin.org</title>'
```

Програма успішно отримала HTML-вміст сторінки через HTTP-запит.

## 4. Перевірка коду за допомогою flake8

Для перевірки стилю коду було встановлено `flake8`:

```bash
pipenv install --dev flake8
```

Після запуску:

```bash
pipenv run flake8
```

Було знайдено наступні помилки:

```bash

./pythonfile.py:28:1: E302 expected 2 blank lines, found 1
./pythonfile.py:36:1: E305 expected 2 blank lines after class or function definition, found 1
./pythonfile.py:39:1: W391 blank line at end of file
```

### Виявлені проблеми:

* невикористаний імпорт;
* недостатня кількість порожніх рядків;
* зайвий пустий рядок у кінці файлу.

Після виправлення помилок повторна перевірка пройшла успішно.

## 8. Робота з Pipenv

Було створено віртуальне середовище через `pipenv`:

```bash
pipenv --python 3.14
pipenv install requests
```

Після цього автоматично створились файли:

### `Pipfile`

Містить:

* залежності проєкту;
* версію Python;
* джерела пакетів.

### `Pipfile.lock`

Містить:

* точні версії усіх бібліотек;
* хеші пакетів;
* дерево залежностей.

## 11. Робота зі змінними середовища

Було створено файл `.env`:

```env
IT_TEST=HelloWorld
```

та Python-файл:

```python
import os
print(f"Значення змінної IT_TEST = {os.environ['IT_TEST']}")
```

### Результат

При запуску через `pipenv` змінна середовища успішно зчитувалась.

### Якщо запускати без віртуального середовища

Без активації середовища або без `pipenv run` змінна `IT_TEST` не буде знайдена, через що виникне помилка:

```python
KeyError: 'IT_TEST'
```
